Give list tags with a child accordion the class "accordion-inside". Only one child accordion per list tag is permitted. The child accordion has to be a direct child of the list tag.
Mark the elements which should open the child accordion with the class "accordion-trigger". Only the list element itself and direct child tags will work.
To have a child accordion opened at the initialisation give its list tag the class "opened".





<div class="input-field select">
    <select placeholder="{% trans 'Please choose' %}" size="4">
        <option>--------</option>
        <option value="option-1">Option 1</option>
        <option value="option-2">Option 2</option>
        ...
        <option value="option-8">Option 8</option>
    </select>
</div>

Always add an empty option on top of all options like "<option>--------</option>".
Only the first option is permitted to have no or an empty value!!!
Use the "size" attribute to define the height of the dropdown in amount of visible options. Default is 5.
Use the event "closed" to catch the moment when the user made a choice and closed the dropdown. Only then the select box has the final value.

Always add an empty option on top of all options like "<option>--------</option>". Even if the select box is required. You have to check afterwards if the value of the select box is empty which means the user never touched the select box.

Autoload options

<div class="input-field select">
    <select class="autoload" placeholder="{% trans 'Please start typing' %}" data-load-url="http://get/options.de" data-load-start="3">
        <option>--------</option>
    </select>
</div>

Loads the options from an external source depending on the value inputed into the field.
Define in "data-load-url" the path to the script which provides the values.
Define in "data-load-start" the amount of letters needed to be inputed to start loading the options. Default is 2.
Functionality needs still to be implemented.
